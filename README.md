# 📺 Dynamic Digital Signage Player & Remote QR Controller

Welcome to the **Zero-Build Dynamic Digital Signage Player**! This project provides a robust, lightweight, and aspect-ratio-perfect solution for displaying digital campaigns with a seamlessly integrated, remote-controlled QR code feature.

---

## 🚀 Core Functionality

At its heart, this is a **zero-build static site** (`index-fixed.html`, `index-fixed.js`) designed for digital signage. 
It renders a meticulously scaled `iframe` containing campaign layouts without introducing any aspect ratio distortion (no "black bands" or letterboxing). The display mathematically calculates the available screen space and intelligently scales the content to fit perfectly while maintaining its intended proportions.

## ✨ Key Features

1. **Perfect Aspect Ratio Scaling:** Eliminates distorted resolutions or letterboxing. Campaign layouts look exactly as intended on any screen size.
2. **Floating QR Code Overlay:** A new, toggleable QR code floats neatly in the bottom-right corner of the display. This QR code is generated dynamically on the client side using the lightweight `qrcode.js` library via CDN.
3. **Fully Dynamic Data Binding:** The site runs entirely off URL Query Parameters, meaning no server-side compilation is required to launch new campaigns.
4. **Zero-Build Architecture:** No `npm run build` or Webpack needed. It's pure, modern HTML, CSS, and Vanilla JavaScript.

## 🏗️ Architecture & How It Works

The architecture is entirely client-side and dynamically controlled by URL parameters. 

When the player loads, `index-fixed.js` parses the browser's URL query string for two critical parameters:
*   `campId` (Campaign ID)
*   `uuid` (Unique Identifier for the device/session)

**Dynamic Routing:**
*   **Iframe Source:** The `campId` is injected into the iframe URL to load the specific campaign content:
    `https://samcmsuser.s3.amazonaws.com/camp/{campId}/index.html`
*   **QR Code Target:** The `campId` and `uuid` are combined to generate the URL that the QR code points to (a remote controller interface for the end-user's phone):
    `https://www.runsam.com/layouts/rc/{campId}/{uuid}`

*(Note: If no parameters are provided in the URL, the system gracefully falls back to default values to ensure the display never breaks.)*

## 🛠️ Usage Guide

To use the player, simply append the desired `campId` and `uuid` to your deployment URL.

**URL Structure:**
```
https://<your-domain>/index-fixed.html?campId=<YOUR_CAMP_ID>&uuid=<YOUR_UUID>
```

**Example:**
If your campaign ID is `12345` and the screen's UUID is `abc-987`, the URL would be:
```
https://<your-domain>/index-fixed.html?campId=12345&uuid=abc-987
```
Upon loading this URL:
1. The iframe will display the campaign from the AWS S3 bucket for `12345`.
2. The QR code will generate in the bottom-right corner, pointing mobile users to `https://www.runsam.com/layouts/rc/12345/abc-987`.
